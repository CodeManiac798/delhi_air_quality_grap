# Archive

Historical planning and process documents from earlier phases of the project.
Each file is a snapshot in time — dated, and superseded by the finished
project described in the [root README](../../README.md). They are kept for
engineering-process history, not as current reference material.

| File | What it was |
|---|---|
| [`project_log.md`](project_log.md) | Early development log: data-source scouting, station selection, Phase 1 build notes. |
| [`issue_tracker.md`](issue_tracker.md) | Engineering issue log as of the start of Phase 2 (5 GRAP events, pre-analysis). |
| [`project_status_report.md`](project_status_report.md) | Completion snapshot (~35%) at the end of Phase 2 preparation. |
| [`phase2_analysis_checklist.md`](phase2_analysis_checklist.md) | Forward-looking task list written before the analysis notebooks existed. |
| [`powerbi_data_model.md`](powerbi_data_model.md) | Original semantic-model sketch, superseded by [`docs/powerbi_architecture.md`](../powerbi_architecture.md). |

For current documentation, start at [`docs/README.md`](../README.md).
